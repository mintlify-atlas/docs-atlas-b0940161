## Overview

The `TaskResponse` model represents a single preparation task assigned to a kitchen station. Tasks are automatically created when orders are processed and distributed to the appropriate station based on product types.

## Fields

<ResponseField name="id" type="long" required>
  Unique identifier for the task.
</ResponseField>

<ResponseField name="orderId" type="long" required>
  The ID of the order this task belongs to. Multiple tasks can belong to the same order if products need to be prepared at different stations.
</ResponseField>

<ResponseField name="station" type="string" required>
  The kitchen station responsible for this task.
  
  **Possible values:**
  - `BAR` - For drinks
  - `HOT_KITCHEN` - For hot dishes
  - `COLD_KITCHEN` - For cold dishes
</ResponseField>

<ResponseField name="tableNumber" type="string" required>
  The table number this task is for. Used to deliver the prepared items to the correct table.
</ResponseField>

<ResponseField name="products" type="array" required>
  Array of products included in this task. All products in a task are prepared at the same station.
  
  <Expandable title="Product object properties">
    <ResponseField name="name" type="string" required>
      The name of the product to prepare (e.g., "Caesar Salad", "Mojito")
    </ResponseField>
    
    <ResponseField name="type" type="string" required>
      The product type: `DRINK`, `HOT_DISH`, or `COLD_DISH`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string (ISO 8601)" required>
  Timestamp when the task was created.
  
  Format: `YYYY-MM-DDTHH:mm:ss`
</ResponseField>

<ResponseField name="status" type="string" required>
  Current status of the task.
  
  **Possible values:**
  - `PENDING` - Task is waiting to be started
  - `IN_PREPARATION` - Task is currently being prepared
  - `COMPLETED` - Task has been finished
</ResponseField>

<ResponseField name="startedAt" type="string (ISO 8601)" optional>
  Timestamp when preparation started. Will be `null` if the task hasn't been started yet.
  
  Format: `YYYY-MM-DDTHH:mm:ss`
</ResponseField>

<ResponseField name="completedAt" type="string (ISO 8601)" optional>
  Timestamp when the task was completed. Will be `null` if the task hasn't been completed yet.
  
  Format: `YYYY-MM-DDTHH:mm:ss`
</ResponseField>

## Task Lifecycle

<Steps>
  <Step title="Task Created">
    Task is created with status `PENDING` when an order is submitted. The `createdAt` timestamp is set.
  </Step>
  
  <Step title="Preparation Started">
    When a kitchen worker starts preparing the task, status changes to `IN_PREPARATION` and `startedAt` is set.
  </Step>
  
  <Step title="Task Completed">
    When preparation is finished, status changes to `COMPLETED` and `completedAt` is set.
  </Step>
</Steps>

  "status": "PENDING",
  "startedAt": null,
  "completedAt": null
}
